#### 래스터 그래픽스(Raster Graphics)
- 디스플레이가 2차원 격자 형태의 그림 요소(픽셀pixel)을 갖고 있음을 뜻함
- 픽셀들의 색상과 세기의 결합을 통해, 디스플레이를 보는 사람은 이를 연속적인 이미지로 지각
- 래스터 디스플레이의 해상도(resoulution)는 픽셀 격자의 너비와 높이를 나타냄
	- 1920x1080의 해상도는 1080개의 행이 있고 각 행에는 1920개의 픽셀을 포함

#### 색상 버퍼(Color buffer)
- 전체 화면의 색상 정보를 포함하고 있는 메모리상의 위치
- 색상 버퍼의 메모리 사용률은 각 픽셀이 사용한 비트의 수에 의존
- 일반적인 24비트 색상 깊이(color depth)에서 빨강, 녹색, 파랑은 각각 8비트를 사용
	- 이는 2^24 또는 16,777,216개의 독특한 색깔을 표현할 수 있다는 의미
- 프레임을 위한 색상 데이터를 포함하는 메모리 위치를 표현하기 위해 프레임 버퍼(frame buffer)를 사용하기도 함
- 프레임 버퍼의 가장 정확한 정의는 색상 버퍼와 기타 버퍼(깊이, 스텐실 등)의 조합


더블 버퍼링 그 자체만으로는 스크린 테어링을 제거하지 못함
스크린 테어링은 게임이 버퍼 X로 쓰기 시작하려고 할 때, 디스플레이가 버퍼 X를 그리고 있는 중이라면 여전히 발생
이 상황은 오직 게임이 너무 빨리 업데이트되는 경우에만 일어남
문제에 대한 해결책은 디스플레이가 버퍼를 교환하기 전에 자신이 사용하고 있는 버퍼를 모두 그릴 때까지 게임이 기다리는 것
다시 말해 게임이 버퍼를 교체하길 원할 때 디스플레이가 여전히 버퍼 X를 그리고 있다면
게임은 디스플레이가 버퍼 X를 다 그릴 때까지 기다려야 함.
이 접근법을 수직 동기화(verical synchronization) 또는 화면을 새로 갱신하려 할 때 모니터가 보내는 신호의 이름을 따서 vsync라 부름

수직 동기화를 사용하면 프레임레이트가 떨어질 수 있음
최근에는 적응적 화면 재생 빈도(adaptive refresh rate)로 이 딜레마에 대한 해결책을 찾고 있음

#### 모놀리식 클래스(Monolithic Class)
- 게임 객체 모델 접근법 중 하나로 모든 게임 객체가 하나의 기본 클래스를 상속하기 때문에 표준 객체지향 클래스 계층 구조로 게임 객체를 선언하는 것
- 단점 : 모든 게임 객체가 기본 게임 객체의 모든 속성과 기능을 가져야 함
	- 예를 들어, 모든 액터가 자신을 갱신하거나 그린다고 가정하면, 화면에 보이지 않는 객체도 Draw를 호출하여 시간 낭비가 됨
	- 상속에 따른 문제점은 게임의 기능이 많아짐에 따라 보다 명확해짐

#### 컴포넌트 기반 게임 객체 모델
- 게임 객체 클래스는 존재하지만, 게임 객체의 서브 클래스는 없음. 게임 객체 클래스는 필요에 따라 기능을 구현한 컴포넌트 객체의 컬렉션을 가짐
- 유니티 게임 엔진이 사용
- 컴포넌트 객체 모델을 구현하는 한 가지 방법은 컴포넌트를 기반으로 하는 클래스 계층도를 가지는 것
- 특정 기능이 필요한 게임 객체에만 해당 기능을 구현한 컴포넌트를 추가하면 된다는 장점
- 순수 컴포넌트 시스템은 게임 객체 컴포넌트들 간의 의존성이 명확하지 않다는 단점
- 언리얼 엔진의 경우 모놀리식 계층 구조와 컴포넌트 객체 모델을 섞은 하이브리드 형태
	- 컴포넌트 간의 통신 오버헤드를 완전히 제거하지는 못하지만, 어느 정도 피할 수 있음
	- 모든 액터는 변환 데이터와 같은 중요 속성을 갖고 있기 때문

#### 스프라이트(Sprite)
- 캐릭터나 배경, 기타 동적인 물체를 나타내는 데 사용되는 일반적인 2D 게임상의 시각적 오브젝트
- 모바일 게임의 경우 스프라이트 데이터가 게임 전체의 다운로드 크기를 차지
- 각 스프라이트는 스프라이트와 관련된 이미지 파일을 하나 이상 갖고 있음
- 플립북 애니메이션(Flipbook Animation) 같은 테크닉을 사용해서 스프라이트 애니메이션을 구현
	- 빠르고 연속적으로 일련의 정적인 2D 이미지를 넘겨서 움직임의 환영을 만들어내는 애니메이션 기법
	- 보통 24FPS의 사용을 선택하며 격투 게임과 같은 일부 장르에서는 60FPS 사용

#### 화가 알고리즘(Painter's Algorithm)
- 게임은 스프라이트를 뒤쪽에서 앞쪽순으로 그림
- 배경을 먼저 그리고 캐릭터를 그리는 순서
